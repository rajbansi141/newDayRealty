STEP 6: USER SERVICE
=====================

File Created: frontend/src/services/userService.js
Location: d:\newDayRealty\frontend\src\services\userService.js

Purpose:
--------
Handle admin operations for user management.
All methods require admin authentication.

Methods Implemented:
--------------------

1. getUsers(filters = {})
   - Fetches all users with optional filters
   - Filters: role, status, page, limit
   - Returns: { success, data: [users], count, total, pagination }

2. getUser(id)
   - Fetches single user by ID
   - Includes user's properties and property count
   - Returns: { success, data: { user, properties, propertyCount } }

3. getUserStats()
   - Fetches user statistics for dashboard
   - Returns: { success, data: { totalUsers, activeUsers, adminUsers, etc. } }

4. updateUser(id, userData)
   - Updates user information
   - Can update: name, email, phone, address, role, isActive
   - Returns: { success, data: user }

5. deleteUser(id)
   - Deletes user and all their properties
   - Permanent action - cannot be undone
   - Returns: { success, message }

6. toggleUserStatus(id)
   - Activates/deactivates user account
   - Toggles isActive field
   - Returns: { success, message, data: user }

Usage Examples:
---------------

import userService from '../services/userService';

// Get all users (admin dashboard)
const { data: users } = await userService.getUsers({
  role: 'user',
  status: 'active',
  page: 1,
  limit: 10
});

// Get user statistics
const { data: stats } = await userService.getUserStats();
console.log('Total users:', stats.totalUsers);
console.log('Active users:', stats.activeUsers);
console.log('Recent users:', stats.recentUsers);

// Get single user with properties
const { data } = await userService.getUser('123');
console.log('User:', data.user);
console.log('Properties:', data.properties);
console.log('Property count:', data.propertyCount);

// Update user
await userService.updateUser('123', {
  name: 'Updated Name',
  role: 'admin'
});

// Toggle user status (activate/deactivate)
await userService.toggleUserStatus('123');

// Delete user
if (confirm('Are you sure?')) {
  await userService.deleteUser('123');
}

Filter Options:
---------------
- role: 'user' | 'admin'
- status: 'active' | 'inactive'
- page: number (for pagination)
- limit: number (items per page)

User Statistics Response:
--------------------------
{
  totalUsers: number,
  activeUsers: number,
  inactiveUsers: number,
  adminUsers: number,
  regularUsers: number,
  recentUsers: number (last 30 days)
}

Security Notes:
---------------
1. All methods require admin authentication
2. Backend validates admin role before processing
3. Non-admin users will receive 403 Forbidden error
4. Token must be valid and not expired

Benefits:
---------
1. Complete user management for admin dashboard
2. Statistics for analytics
3. Bulk operations support (via filters)
4. Safe deletion (removes user's properties too)
5. Easy status toggling
