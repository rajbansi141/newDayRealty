STEP 2: API CONFIGURATION FILE
===============================

File Created: frontend/src/config/api.js
Location: d:\newDayRealty\frontend\src\config\api.js

Purpose:
--------
Centralize all API endpoints and provide helper functions for making API requests.

Key Components:
---------------

1. API_URL
   - Base URL from environment variables
   - Fallback to localhost:5000/api if not set

2. API_ENDPOINTS
   - Organized by feature (AUTH, PROPERTIES, CONTACT, USERS)
   - All endpoints in one place for easy maintenance
   - Dynamic endpoints using functions (e.g., SINGLE: (id) => ...)

3. getAuthToken()
   - Retrieves JWT token from localStorage
   - Used for authenticated requests

4. getAuthHeaders()
   - Returns headers object with Content-Type and Authorization
   - Automatically includes token if available

5. apiFetch()
   - Wrapper around native fetch API
   - Automatically adds auth headers
   - Handles errors consistently
   - Returns parsed JSON data

Usage Examples:
---------------

// Import in your components
import { API_ENDPOINTS, apiFetch } from '../config/api';

// Get all properties
const data = await apiFetch(API_ENDPOINTS.PROPERTIES.BASE);

// Get single property
const property = await apiFetch(API_ENDPOINTS.PROPERTIES.SINGLE(123));

// Create property (authenticated)
const newProperty = await apiFetch(API_ENDPOINTS.PROPERTIES.BASE, {
  method: 'POST',
  body: JSON.stringify(propertyData)
});

Benefits:
---------
1. Single source of truth for all API endpoints
2. Easy to update URLs when backend changes
3. Consistent error handling
4. Automatic authentication header injection
5. Type-safe endpoint generation with functions
